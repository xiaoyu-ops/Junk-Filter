🔑 核心设计特点
================================================================================

【1】依赖注入（Dependency Injection）
✅ 所有依赖通过参数传入（接口注入）
✅ 依赖于接口而非具体实现
✅ 便于单元测试（mock 实现）
✅ 便于替换实现（Strategy 模式）

【2】接口隔离（Interface Segregation）
✅ 每个模块定义自己的接口
✅ 清晰的业务契约（domain/interfaces.go）
✅ 支持多个实现共存
✅ 便于扩展和测试

【3】工厂模式（Factory Pattern）
✅ 统一的对象创建（service/factory.go）
✅ 集中管理依赖关系
✅ 自动初始化顺序
✅ 统一的资源关闭

【4】配置驱动（Configuration-Driven）
✅ 默认值 → YAML → 环境变量
✅ P0 优化值集中管理
✅ 无需重新编译
✅ 支持多环境部署

💡 关键设计原则
================================================================================

1. SOLID 原则应用
   S - 单一职责：每个模块一个职责
   O - 开闭原则：对扩展开放，对修改关闭
   L - Liskov 替换：实现可互相替换
   I - 接口隔离：清晰的最小接口
   D - 依赖反转：依赖于抽象，不依赖于具体

2. 设计模式应用
   - Factory Pattern: service/factory.go
   - Dependency Injection: 构造函数参数
   - Strategy Pattern: 通过接口实现
   - Adapter Pattern: 基础设施封装

3. P0 优化应用
   ✅ 数据库连接池: 50 个连接
   ✅ Worker 数: 20 个（从 5 改为）
   ✅ 超时: 30 秒（从 10s 改为）
   ✅ 抓取间隔: 30 分钟（从 1h 改为）
